# 子分类功能说明

## ✅ 确认：子分类是级别的下一级

是的，子分类确实是级别（如RG）的下一级分类。

**分类层级结构：**
```
级别 (Grade)
  └── 子分类 (Subcategory)
      ├── 普通版
      ├── 网络限定版
      ├── 其他限定版
      ├── EVANGELION系列
      ├── 勇者王系列
      └── 参考出品/开发中
```

例如：
- **RG级别** 下有：
  - 普通版（43款）
  - 网络限定版（78款）
  - 其他限定版（81款）
  - EVANGELION系列（14款）
  - 勇者王系列（2款）
  - 参考出品/开发中（3款）

## 📊 数据库结构

- `grade`: 级别（RG, MG, HG等）
- `subcategory`: 子分类（普通版、网络限定版等）

## 🔄 更新流程

1. ✅ **数据库已更新** - subcategory字段已添加
2. 🔄 **正在重新爬取数据** - 会自动识别并保存子分类
3. ✅ **完成后可在网页中筛选**

## 📝 使用说明

爬取完成后，在网页应用中：

1. 访问"高达列表"
2. 选择级别（如RG）
3. 子分类下拉框会自动显示该级别的所有子分类
4. 选择子分类进行筛选

## 💡 提示

- 子分类信息会在爬取时自动识别
- 如果某个模型没有子分类，会默认为"普通版"
- 可以同时按级别和子分类进行筛选
